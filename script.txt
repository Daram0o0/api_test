예상 추가 기능 및 고려해야 할 점

1. 해당 주소 + 불특정 URL을 열고, 거기로 진입시 해당 학교가 보여야 하며, 
2. 비밀번호를 입력해서 맞아야 볼 수 있어야 함.
3. MOCK데이터(?)와 함께 볼 수 있어야 함.
4. 데이터 값을 추가 가능해야 함.
5. 테이블이 이쁘면 좋겠징...?
6. 학교를 입력 가능하게 하면 안됨. 고객사외의 학교들이 보일 수 있음.

문제상황
1. api.moaform.com/v1/forms
    - 설문 목록 가져오기 에서 collection.responses_count가 예상한 수와 다르게 나옴
2. api.moaform.com/v1/forms/{form_id}/responses
    - 여기서 total_items가 결과 목록의 전체 개수임

해결 방안
1. 1번에서 api 호출하면서 원하는 key들을 수집해 배열?객체를 만들어 둠.
2. 2번 api를 map돌려서 호출하며 하나씩 값을 찍어냄.

현재 진행상태
"한국영상대.txt"폴더를 참고해주시면 감사하겠습니다.
